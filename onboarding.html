Layout

The layout of the FTUE screens is pretty standard and true to the platform.

Splash Screens

Splash/ Launch screen is the screen the user sees upon launch of the app while they wait for the app to load. The splashscreen isn’t an opportunity for artistic expression. It’s solely intended to enhance the perception of your app as quick to launch and immediately ready for use. Every app must supply a launch screen.

  • The splash screen is only shown on kill-launch sequences and when it takes time to load the app.
  • If the app loading time exceeds 1 second then a loading indicator should be shown.

Reach out to Yong for splash screen mocks and specs with and without loading indicator

All other screens

All the remaining screens in the FTUE have a standard structure.

  • The app bar lies at the top and is optional in screens that need a title.
  • The page content folows below it.

Buttons

There are primary and secondary buttons that will be used all thorugh the FTUE

  • Primary buttons will exhibit the primary app color
  • Secondary buttons will be shown as plain text, so that the user is not distracted by them.
  • Primary buttons will appear only after the required/ nessary fields have been filled.

Loading indicators

Loading indicators are shown when the user interacts with the app and the app takes more than 1 second to respond. These can be determinate or indeterminate

  • When indicators are determinate they indicate how long an operation will take when the percentage complete is detectable.
  • When indicators are indeterminate they request that the user wait while something finishes when it’s not necessary to indicate how long it will take.
  • If the apps processing time exceeds 1 second then a loading indicator must be shown.

Password fields

Password fields are similar to text fields. There are a few exceptions listed below however.

  • Password fields must have a show and hide option
  • Password fields must be accompanied with requirements shown one at time to the user. For instance if there are 4 requirements, show requirement 1 first. Once that is fulfilled show 2, then 3 and then 4.
  • Password text must be hidden by default
